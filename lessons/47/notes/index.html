Notes for the lesson 47. Project 3 guidelines.

General requirements.

For the project 3 we need to make responsive web app.

There are several options of the websites we can select:

  • OpenWeatherApp - user should be able to load the weather forecast for specific location. It can be filtered by 3 and 5 days. API.
  • The Movie Database search movies, use year, janre, rating as filters. Use modal to show detailed info about the movie. User should be able to use light or dark theme. API.
  • Wallhaven - user should be able to find wallpapers with the request. There should be filters: sorting, order and permissions. Results of the search should show up thumbnails. The image is clicked it should show up in the lightbox. API.
  • News API - user should be able to see the latest news and sort them out using filters: by phrase, category selection, country selection. Header should include categories that will load 50 articles. Shoukd have a theme switcher. API.
  • Marvel Library. API.
    • Show all the characters, when user opens the page.
    • User should be able to search for the character
    • Download user info and show it in modal or in the other page.
    • Shoukd have a theme switcher.
  • Musixmatch - app for songs tests. User should be able to find a track by request. List the results on the page. When one result is selected, the name of the track and the text of the song should be showm. Shoukd have a theme switcher. API.

Requirements:

  • Should have favicon.
  • Should be responsive.
  • Should use .scss and can use frameworks.
  • Should have valid css and HTML.
  • Semantic code.
  • Should be on host (vercel).
  • Speed should be optimzed to 85% on mobile and desktop.

Marvel Library.

node_modules - are already set to be in .gitignore.

React has some files with .pnp or .pnp.jsextention, they are are also in gitignore and needed for react.

The folder /build is the same as we had /dist, where all compiled things are added.

React also might include .env files.

Folder /public includes: robots.txt, favicon (can remove or change to my own), manifest for favicon (can remove or change to my own), logo (can remove or change to my own), and index.html (need to change to my own).

Folder /src includes: all our styles and components.

  • File App.js is the main file for our application.
    • We import images via import something from './someplace'
    • We import css via import './App.css'. This is one of the options, we will check more later.
    • File App.css includes the styles for our front page.
  • File App.test.js is the file for testing.
  • File setupTests.js is for testing parameters.
  • File index.js is entering point for our application. We initialize React, ReactDOM, reportWebVitals, and App from here.
  • File index.css includes the global styles. Best practices are that this file is removed and the separate file with styles for body, html or tags added.
