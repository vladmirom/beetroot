Advanced SCSS

  • It's better not to use the calc() css function.
  • In native calculation (+/-) in scss we can calculate only the same data types. (px - px, % - %, rem - rem)
  • We can multiply any data types except pixels. In this case we can use just number: 10px * 10.
  • Division: just / sign is deprecated. The division should be used in (). (100px / 2);. Also, if we divide the same data types, they will give the number without the datatype. In this case we just add the datatype additionally: (100px / 2px) + px;. Different data types don't work.
  • We can create mixins with local variables and add the default values to them. It is useful, because if we forget the value hwen we cal the mixin, the error occurs.
  • We can have mixin in mixin, or a group of mixins in one mixin.
  • When we call a mixin, we can overwrite the variable. It is useful, when we have a mixin wiht 5 different variables, but when we call the mixin, we want to pass only 1 value.
  • It is better to write variables without default value first and then with default values in the end.
  • We can have the mixin that shows different content. Instead of calling with just () we call mixin with () and {}. Inside of {} we put css rules that we want to pass. In the mixin declaration we need to use @content in the place where we want to paste the content.
