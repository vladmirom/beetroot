Notes for the lesson 7. CSS selectors.

General selectors

  1. * star selector selects all elements on the webiste. Useful to clear and prepare your css for the further work. It also can be used in nesting (for instance, we don't have access to DOM structure).
  2. Dont use anything inside the * except margin, padding. Font size, font weight, lineheight should not be set here. Otherwise, the correct nesting will break here.
  3. We can work directly with tags. Not the best practice.
  4. We can use classes and ids in css. Id has bigger specificity, so id will overwrite class.
  5. We don't style id because it is unique for the page, and classes can be reused on the page.
  6. We don't write px to 0.
  7. The level of nesting is defined by space between elements. This code div h3 means to select every h3 element that is located inside the div.
  8. If two classes or elements are added without the space css will look for them as sibings. EG: div .class-red.class-blue means to select the element inside the div that has both class-red class-blue classes.
  9. The code writen in the bottom will overwrite the code in the top and will be used.

General styling

  1. It's better to add font size, weight and line-space in body
  2. Font is set or duplicated in html tag to set rems. The defaut font size is 16px.
  3. line-height is better set in % if font-size is in rem or em.

CSS operators

  1. Nice reference is here: Operators
  2. Arrow > tells to find child element. EG: .some-class > p will select every p element inside the some-class. In this case <div class="some-class"><div><p>some text here</p></div></div> it will not be selected. In this case <div class="some-class"><p><p>some text here</p></p></div> it will be selected, because it inherits father's properties.
  3. Plus + selects all the siblings of the element, except this element.
  4. Tilda ~ selects all the next siblings after this one and not including this one.
  5. Coma , adds properties to both elements written through coma. The main task is to minimize css and DRY.
  6. Box brackets [] lets you work with attributes of the element.
    • a[href] - will select all elements on the webiste that have href element.
    • a[href="#"] - will select all elements on the website that have the exact "#" as the value of the href.
    • a[title~="Lorem"] - will select all elements on the webiste that include in its content "Lorem" (so there might be some other content together with "Lorem") as a value of the title. Note that "Lorem" should have space in html after it.
    • a[href|="http"] - will select all elements on the webiste that include in its content "http" (so there might be some other content together with "http") as a value of the href. Note that after the "http" should go hyphhen "-".
    • a[href$="/"] - will select all elements on the webiste that include in its content "/" as the last element as a value of the href.
    • a[href*="https"] - will select all elements on the webiste that include in its content "https" at least once as a value of the href.
    • a[href^="http"] - will select all elements on the webiste that include in its content starting with "http" as a value of the href.

Pesudo-classes

  1. Selecting the first child with :first-child and the last child with :last-child pseudo-classes.
  2. Selecting the first element with :first-of-type and the last element with :last-of-type pseudo-classes.
  3. Selecting the the children elements inside the listing with the help of :nth-child().
    • :nth-child(even) is selecting all even numbers: 2,4,6.
    • :nth-child(odd) is selecting all odd numbers: 1,3,5.
    • :nth-child(n). There won't be iteration, only 1 element will be selected.. In example :nth-child(2), only the 2d element will be selected.
    • :nth-child(an+b). n starts with 0, so in example :nth-child(2n + 1) in the first iteration the calculation will look like 2*0+1 = 1. The next iteration will be 2*1+1 = 3. The next iteration will be 2*2+1 = 5. And so on...
    • :first-child, :nth-child(1), and :nth-child(0n + 1) all three will have the same result - will select first element.
    • :nth-child(an + 1), and :nth-of-type(an + 1) will start the iteration calculation from the beginning (or top). :nth-last-child(an + 1), and :nth-last-of-type(an + 1) will start the iteration calculation from the ending (or bottom).
  4. :hover styling the hover of anything (not only the link).
  5. :visited styling the visited of the link.
  6. :link styling the not yet visited of the link.
  7. :focus styling the selected focused element.
  8. :active styling the selected element.
  9. :checked styling the checked checkbox.
  10. :empty represents any element that has no children.

Pseudo-elements

  • ::first-letter applies styles to the first letter of the first line of a block-level element, but only when not preceded by other content (such as images or inline tables).
  • ::marker styling the marker of the list. Can style both <ol> and <ul>. It should be applieed to <li>
  • ::before creates a pseudo-element that is the first child of the selected element. Works only with binary tags (that have closing tag). It is important to add display. We can also add to content link to the image content: url('./assets/img.png'); or nothing content: "";.
  • ::after creates a pseudo-element that is the last child of the selected element. Works only with binary tags (that have closing tag). It is important to add display. We can also add to content link to the image content: url('./assets/img.png'); or nothing content: "";.

    • Float

  • Better not to use. It's better to use flex and grid instead.
  • Works badly with position: absolute;.
  • Always sets element to be a block, except flex. Flex remains flex.
  • Need to use clear: both; on the parent element, so it the content after is positioned correctly.
  • Margins are calculated differently if float is used.
