Why we use React.

React creates it's own virtual DOM and makes changes in it first, then it checks the changes in the virtual DOM and after that it uploades changes to the real DOM. This gives us the opportunity to do things faster, or inject the changes to the specific elements in arrays, or objects.

We can make changes to the specific elements. This means that we don't re-load the whole page, but just a specific changed element, like section, or button.

Another advantege of React is the easines or creating SPA (Single Page Application).

Getting started.

Starting the project goes with the command in the empty folder in terminal npx create-react-app ..

If we want to create a subfolder in our folder with the react project, we run the command in our folder, where we want to have a subfolder: npx create-react-app subfolder-name.

NPX is Node Package Executor - we want to install something just for 1 execution.

For tests we use Jest.

To start project we need to use npm start and to run tests npm test. Also we can stop npm with the npm stop command.

React uses webpack in it's core.

Structure.

node_modules - are already set to be in .gitignore.

React has some files with .pnp or .pnp.jsextention, they are are also in gitignore and needed for react.

The folder /build is the same as we had /dist, where all compiled things are added.

React also might include .env files.

Folder /public includes: robots.txt, favicon (can remove or change to my own), manifest for favicon (can remove or change to my own), logo (can remove or change to my own), and index.html (need to change to my own).

Folder /src includes: all our styles and components.

  • File App.js is the main file for our application.
    • We import images via import something from './someplace'
    • We import css via import './App.css'. This is one of the options, we will check more later.
    • File App.css includes the styles for our front page.
  • File App.test.js is the file for testing.
  • File setupTests.js is for testing parameters.
  • File index.js is entering point for our application. We initialize React, ReactDOM, reportWebVitals, and App from here.
  • File index.css includes the global styles. Best practices are that this file is removed and the separate file with styles for body, html or tags added.

Naming files rules.

There are three most popular ways to name the files. The first one is the most popular.

  • camelCase - for js files and image, not css. And always PascalCase for components.
  • kebab-case - for js files and image, not css. And always PascalCase for components.
  • PascalCase - for js files and image, not css. And always PascalCase for components.
