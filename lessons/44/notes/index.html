Why we use React.

React creates it's own virtual DOM and makes changes in it first, then it checks the changes in the virtual DOM and after that it uploades changes to the real DOM. This gives us the opportunity to do things faster, or inject the changes to the specific elements in arrays, or objects.

We can make changes to the specific elements. This means that we don't re-load the whole page, but just a specific changed element, like section, or button.

Another advantege of React is the easines or creating SPA (Single Page Application).

Getting started.

React documentation.

Starting the project goes with the command in the empty folder in terminal npx create-react-app ..

If we want to create a subfolder in our folder with the react project, we run the command in our folder, where we want to have a subfolder: npx create-react-app subfolder-name.

NPX is Node Package Executor - we want to install something just for 1 execution.

For tests we use Jest.

To start project we need to use npm start and to run tests npm test. Also we can stop npm with the npm stop command.

React uses webpack in it's core.

Structure.

node_modules - are already set to be in .gitignore.

React has some files with .pnp or .pnp.jsextention, they are are also in gitignore and needed for react.

The folder /build is the same as we had /dist, where all compiled things are added.

React also might include .env files.

Folder /public includes: robots.txt, favicon (can remove or change to my own), manifest for favicon (can remove or change to my own), logo (can remove or change to my own), and index.html (need to change to my own).

Folder /src includes: all our styles and components.

  • File App.js is the main file for our application.
    • We import images via import something from './someplace'
    • We import css via import './App.css'. This is one of the options, we will check more later.
    • File App.css includes the styles for our front page.
  • File App.test.js is the file for testing.
  • File setupTests.js is for testing parameters.
  • File index.js is entering point for our application. We initialize React, ReactDOM, reportWebVitals, and App from here.
  • File index.css includes the global styles. Best practices are that this file is removed and the separate file with styles for body, html or tags added.

Naming files rules.

There are three most popular ways to name the files. The first one is the most popular.

  • camelCase - for js files and image, not css. And always PascalCase for components.
  • kebab-case - for js files and image, not css. And always PascalCase for components.
  • PascalCase - for js files and image, not css. And always PascalCase for components.

React components.

We can divide different parts of the page into components. Components can include other components without any limitations.

The idea of dividing pages into components is to make some parts of the website reusable, therefore these components should be independent from one another.

The common practice is to have ccomponents in separate files with the same name (App.js, App.scss, App.test.js)

The better practice is to separate js files from css. This is Beetroot projects (and default React) structure:

  • node_modules
  • public
  • src
    • assets
      • Button.scss
      • Link.scss
    • components
      • Button.js
      • Link.js
    • App.css
    • App.js
    • App.test.js
    • indext.css
    • index.js
  • .gitignore
  • package-lock.json
  • package.json
  • README.md

    • Link

        • Link.module.scss

      • Link.module.scss

Components should always be named with the Capital letter. React accepts files that start with the lovercase letter as tags.

Components naming is done in PascalCase.

Compontents types

Historically there were 2 types of components formed: Class and Functional components.

In the functional approach, which is the most comon and React itself is trying to use more, the main function name of the component should be eexactly the same as the name of the file.

React syntax.

React used JSX.

JSX is written inside return.

If we have only one line to return, it can go without () return <h1>Title</h1>, but if there are several lines of code, we should wrap it ().

Function can return only 1 element. So we eitheer should wrap everything in <div> or any other wrapper tag. Also, we can use <> some multiline code here </> if we don't want to create a new div in the DOM structure.

Classes in JSX are added with the attribute className: <div className="foo">Some content</div>. The same happens to tabindex.

There's another method to create html structure in the components without JSX. But it's not not used much. It can be done via createElement method.

Constructing React app.

Let's say we have a slider component (in practicing/components/Slider.js). We need to import it first to the App.js file.

We run import on the top of the document import Slider from './components/Slider';.

If we import a big component, it is imported this way import React from 'react', if we importing some method, it should be done with {} import { createElement } from 'react'.

We add slider to the page inside App function's return: <Slider></Slider> or as a shorter version without any parameters: <Slider/>

It is important to export our function: export default Slider;. It will help us to define what is the main function in the doc, especially when we have many functions.

The parent component should not know about child component.
